# Phase 3: Network Basics and TCP Practice

## やったこと

Phase 3 では、Raspberry Pi 上でネットワークの状態を実際に観察し、最後に Python で最小の TCP サーバー / クライアントを実装して通信を確認した。

主に行ったことは以下。

- `ip a` でネットワークインターフェースと IP アドレスを確認
- `ip r` で routing table と default gateway を確認
- `ss -tulpn` で待受ポートと公開範囲を確認
- `ping -c 4 8.8.8.8` で IP アドレス宛ての疎通確認
- `ping -c 4 google.com` で DNS 名前解決を含む疎通確認
- `dig google.com` で DNS 応答の中身を確認
- Python で最小の TCP サーバー / クライアントを作成
- `127.0.0.1` bind 版と `0.0.0.0` bind 版の違いを確認
- localhost 接続と LAN 側アドレス接続の違いを確認

---

## 実行した主なコマンド

```bash
ip a
ip r
ss -tulpn
ping -c 4 8.8.8.8

sudo apt update
```
